## 多线程/并发编程

## 概念

- 原子性

    一个操作，要么成功，要么失败。例如银行转账，两个账号的变动必须要么都成功，要么都失败

- 可见性

    - 多线程访问变量时，一个线程修改变量，其他线程能立即看到变量的修改
    - volatile

- 有序性

    - 代码的执行符合预期。由于编译器和jvm的指令优化，会导致代码编写时的顺序和实际顺序不一致
    - synchronized
    - lock
